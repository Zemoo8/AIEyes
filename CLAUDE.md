# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

AI Eyes (عيون الذكاء) is a mobile accessibility app for visually impaired Arabic speakers in Tunisia. It turns the smartphone into a real-time visual assistant that perceives the world and narrates it back to the user in spoken Arabic.

**Target platform:** Android only (Arabic-first UX, French as secondary language for OCR).

**Stack:** React Native + Expo, New Architecture enabled (`newArchEnabled: true`).

## Core Features

- **Real-time object detection** using YOLOv8 via the phone camera
- **Text recognition (OCR)** for both Arabic and French text
- **Scene description** powered by the Claude vision API for richer, contextual narration
- **Outdoor navigation** via GPS with Arabic audio guidance
- **Indoor navigation** via QR codes placed at university doors and key locations
- **Arabic TTS output** — all feedback is spoken aloud in Arabic

## Architecture Notes

- Single JavaScript codebase via Expo
- Entry point flows into the root component
- Native folders are generated by Expo (prebuild) when needed for custom native modules (likely required for YOLOv8 on-device inference)
- Assets live in the standard Expo `assets/` directory
- `app.json` controls Expo config (permissions for camera, microphone, location, etc. must be declared here)

## Key Integrations

- **YOLOv8** — on-device object detection (likely via a native module or ONNX/TFLite runtime)
- **OCR engine** — Arabic + French text recognition (e.g., ML Kit or Tesseract)
- **Claude vision API** — scene description; requires API key via env var, network connectivity required
- **Expo Speech / native TTS** — Arabic text-to-speech output
- **Expo Location** — GPS for outdoor navigation
- **Expo Camera / BarCodeScanner** — camera feed + QR code scanning for indoor navigation

## Design Principles

- **Audio-first UX:** the interface must be fully usable without sight; every action gives spoken Arabic feedback.
- **Low latency:** object detection and OCR should feel real-time on mid-range Android devices.
- **Offline-capable where possible:** object detection and OCR should work without internet; only scene description (Claude API) requires connectivity.
- **Accessibility first:** large touch targets, high contrast, full TalkBack compatibility, simple gestures.

## Environment Variables

- `ANTHROPIC_API_KEY` (or equivalent Expo public env var) — required for the Claude vision scene description feature.
